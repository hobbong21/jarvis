# Phase 1 — Domain Analysis: SARVIS

> Harness 6 Phase 워크플로우의 Phase 1 산출물. 본 문서는 SARVIS 도메인을
> 에이전트 팀 설계가 가능한 형태로 분해한다.

## 1. 한 줄 정의

**SARVIS** 는 한국어 음성 입출력을 1차 인터페이스로 삼는, 멀티 백엔드 LLM 라우팅·
시각(카메라)·감정·메모리 통합 데스크톱급 어시스턴트 (FastAPI + WebSocket + Vanilla
JS, Replit 5000 포트).

## 2. 사용자 시나리오

| 시나리오 | 빈도 | 지연 예산 | 실패 시 영향 |
|---------|------|---------|------------|
| 마이크로 한국어 발화 → 응답 음성 출력 | 매우 높음 | <2.5s end-to-end | 핵심 가치 손상 |
| 카메라로 얼굴 인식 + 인사 | 중간 | <1.5s | UX 저하 |
| 텍스트 채팅 폴백 (마이크 차단 환경) | 높음 (iframe) | 채팅 수준 | 마이크 막힐 때 유일 경로 |
| 백엔드 전환 (OpenAI / Claude / Ollama) | 낮음 | 즉시 | 비용/가용성 |
| 메모리 기반 멀티턴 | 높음 | <1s 검색 | 맥락 끊김 |

## 3. 도메인 특성

- **단발 vs 반복**: 반복. 같은 사용자가 같은 세션 안에서 수십 번 호출.
- **협업 단위 존재 여부**: 있음 — STT, LLM, TTS, 시각, 메모리, 도구가 매 발화마다
  비대칭으로 협력.
- **결정 지점**: (a) 백엔드 선택, (b) 도구 호출 여부, (c) 시각 컨텍스트 첨부 여부,
  (d) 음성 vs 텍스트 출력.
- **품질 게이트 필요**: 한국어 자연스러움, 응답 길이 (TTS 비용), 부적절 표현 필터.
- **자동 검증 가능 신호**: 한국어 비율, 응답 길이, 도구 출력 스키마, 지연 시간.

## 4. 환경 제약 (Replit)

- 5000 포트 단일 노출, iframe 프록시 → **마이크 게이트 필요** (새 탭 버튼은 구현됨).
- 콜드 스타트 60s 한계 → **OpenCV / Whisper 지연 로딩 필수** (현재 적용됨).
- 백엔드 키: ANTHROPIC 고갈, OPENAI 사용 가능 → 기본 백엔드 OpenAI (현재 적용됨).
- Replit 자동 체크포인트 — 수동 git commit 제한.

## 5. Phase 2 입력 요약

다음 Phase (Architecture Design) 에 전달할 핵심 요건:

1. **반복 호출** — 팀 구성은 영속적, 매 호출마다 재생성하지 않음.
2. **이질적 협업** — 6패턴 중 단일 패턴으로 부족, 합성 필요.
3. **동적 라우팅** — 백엔드/도구/시각 첨부 여부가 입력마다 다름 → Supervisor 필수.
4. **품질 게이트** — TTS 직전 검증 필요 → Generate-Verify 가지 필요.
5. **순차 핵심 경로** — STT → 의도 → LLM → TTS 는 본질적으로 파이프라인.
