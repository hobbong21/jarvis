# S.A.R.V.I.S вАФ Personal AI Assistant

A multimodal AI assistant inspired by the 4-stage agent pattern (Task Planning вЖТ Model Selection вЖТ Task Execution вЖТ Response Generation). Features face recognition, voice interaction, and tool-augmented intelligence.

## Architecture

- **Backend**: FastAPI + WebSockets (`server.py`) on port 5000
- **Frontend**: HTML5 Canvas / Vanilla JS / CSS in the `web/` directory
- **AI Brain**: Claude (Anthropic) or Ollama as LLM backend (`brain.py`)
- **STT**: Faster-Whisper for speech-to-text
- **TTS**: Edge-TTS (Microsoft) for text-to-speech
- **Vision**: OpenCV + optional face_recognition for webcam analysis

## Key Files

- `server.py` вАФ FastAPI web server (entry point for web mode)
- `brain.py` вАФ LLM controller (Claude tool_use loop + Ollama simple chat)
- `tools.py` вАФ Tool definitions and executor (web search, weather, timer, memory, vision)
- `audio_io.py` вАФ Speech recording, Whisper STT, Edge-TTS
- `vision.py` вАФ VisionSystem (desktop) + WebVision (web, browser-pushed frames)
- `config.py` вАФ Centralized configuration via environment variables
- `auth.py` вАФ User authentication with PBKDF2 hashing
- `emotion.py` вАФ 7 assistant emotional states
- `web/` вАФ Frontend assets (HTML, CSS, JS, orb animation)
- `data/` вАФ **All runtime/user data** (`memory.db`, `users.json`, `memory.json`, `faces/`, `harness_actions.jsonl`, `harness_telemetry.jsonl`, `tts_blocklist.json`). The repo's source-tree top level is therefore code-only. Path overridable via env (`SARVIS_MEMORY_DB`, `SARVIS_USERS_FILE`, `SARVIS_FACES_DIR`, `SARVIS_TOOL_MEMORY`); each path's owner module auto-creates parent dirs on first write.

## Environment Variables

- `ANTHROPIC_API_KEY` вАФ Required for Claude backend (set as a secret)
- `OPENAI_API_KEY` вАФ Required for OpenAI backend and `compare` mode (set as a secret)
- `GOOGLE_API_KEY` вАФ Required for Gemini backend (set as a secret). `GEMINI_API_KEY` is honoured as a legacy fallback. Default model `gemini-2.5-flash` (override with `GEMINI_MODEL`).
- `SARVIS_BACKEND` вАФ `"openai"` (default), `"claude"`, `"ollama"`, `"zhipuai"`, `"gemini"`, or `"compare"`
- `OLLAMA_HOST` вАФ Ollama server URL (default `http://localhost:11434`). Use a tunnel URL (e.g. cloudflared) when SARVIS runs on Replit but Ollama runs on a local machine.
- `OLLAMA_MODEL` вАФ Ollama model tag (default `qwen2.5:7b`). Examples: `llama3.2:3b`, `qwen2.5:14b`, `gemma2:9b`.
- `PORCUPINE_ACCESS_KEY` вАФ Optional, for desktop wake-word detection
